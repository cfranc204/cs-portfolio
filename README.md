# cs-portfolio
CS Portfolio – DriverPass Project
The DriverPass project focused on designing a system for a company that helps students prepare for their driving tests. The client wanted a system that would allow users to create accounts, schedule driving lessons, take online practice tests, and manage their progress. The goal was to make everything accessible online and easier to manage compared to a more manual process.
One thing I think I did well was keeping my diagrams clear and easy to follow. Instead of overcomplicating things, I focused on making sure each diagram actually made sense and matched what the system was supposed to do. I also made sure everything connected, like using the same use cases across my activity and sequence diagrams so the design stayed consistent.
If I could go back and revise one part of my work, I would probably spend more time refining my class diagram. While it included the main classes and relationships, I think I could improve it by organizing it even better and possibly adding a little more detail to make the relationships clearer.
When interpreting the user’s needs, I focused on what the client specifically asked for, like online access, scheduling lessons, and practice tests. I made sure those features were reflected in both my diagrams and my written explanations. It’s important to consider the user’s needs because the system is being built for them, not the developer. If the design doesn’t match what the user actually needs, then the system won’t be useful.
When I approach designing software, I like to start by understanding the problem first before jumping into solutions. Breaking things down into smaller steps, like using diagrams, helps me visualize how everything works. In the future, I would continue using tools like UML diagrams and also focus on getting feedback earlier in the process so I can improve the design before it’s finalized.
